# 修复订单列表时间和金额显示问题

## 问题分析

支付页面能正确显示时间和金额，但订单列表显示不正确。对比发现：

1. **支付页面的实现**（正确）：

   - 时间格式化：直接使用 `new Date(timeField)` 然后手动格式化
   - 金额格式化：简单的 `(price / 100).toFixed(2)`

2. **订单列表的问题**：

   - `formatTime` 函数逻辑复杂，可能在某些情况下返回空字符串
   - `formatPrice` 函数有太多检查和日志，可能在某些边界情况下返回 '0.00'
   - 数据映射时可能丢失了 `amount` 或 `createDate` 的值

## 修复方案

### 1. 简化 `formatTime` 函数 (`orders.ts`)

   - 完全参考支付页面的实现方式
   - 直接使用 `new Date(timestamp)` 创建日期对象
   - 手动格式化：`YYYY-MM-DD HH:mm`
   - 移除复杂的判断逻辑，只保留核心功能

### 2. 简化 `formatPrice` 函数 (`orders.ts`)

   - 完全参考支付页面的实现方式
   - 简单的 `(price / 100).toFixed(2)`
   - 移除所有日志和复杂检查

### 3. 确保数据映射正确 (`orders.ts`)

   - 确保 `amount` 和 `createDate` 正确提取为数字类型
   - 如果值为 0，也要保留（因为 0 是有效值）
   - 添加数据验证，确保映射后的数据包含正确的字段

### 4. 优化 WXML 模板 (`orders.wxml`)

   - 确保格式化函数正确调用
   - 移除不必要的条件判断，让格式化函数处理边界情况

## 具体修改

### `miniprogram/pages/orders/orders.ts`

1. **简化 `formatTime` 函数**：
```typescript
formatTime(timeStr: any): string {
  if (!timeStr) return '';
  const d = new Date(timeStr);
  if (isNaN(d.getTime())) return '';
  return `${d.getFullYear()}-${(d.getMonth()+1).toString().padStart(2,'0')}-${d.getDate().toString().padStart(2,'0')} ${d.getHours().toString().padStart(2,'0')}:${d.getMinutes().toString().padStart(2,'0')}`;
}
```

2. **简化 `formatPrice` 函数**：
```typescript
formatPrice(price: number | string | null | undefined): string {
  if (!price && price !== 0) return '0.00';
  const priceNum = typeof price === 'string' ? parseFloat(price) : Number(price);
  if (isNaN(priceNum)) return '0.00';
  return (priceNum / 100).toFixed(2);
}
```

3. **优化数据映射**：

   - 确保 `amount` 和 `createDate` 正确提取
   - 保留 0 值（因为 0 是有效值）
   - 添加数据验证日志

### `miniprogram/pages/orders/orders.wxml`

1. **简化模板绑定**：

   - 创建时间：`{{formatTime(item.createDate || item.orderTime)}}`
   - 订单金额：`{{formatPrice(item.amount)}}`
